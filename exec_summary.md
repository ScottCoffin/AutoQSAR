# AutoQSAR Benchmark Executive Summary

This summary interprets the populated outputs from `portable_colab_qsar_bundle/benchmark_results_summary.ipynb`, using the most recent benchmark run directory:

`benchmark_results/all_benchmarks_no_unimol_20260425_223505`

The notebook is populated with 22 cells and 125 code outputs. The run contains 45 dataset directories, of which 44 have complete metrics, predictions, selected features, and split hashes. The analyzed set is balanced across task types: 22 classification datasets and 22 regression datasets.

## Headline Findings

AutoQSAR is broadly competitive across ADMET, MoleculeNet, Polaris, ChemML, and PODUAM benchmark tasks. Against the updated reference tables generated by the notebook, AutoQSAR is estimated to place in the top 10 on 33 of 37 datasets with available leaderboard or literature references, including 5 apparent rank-1 results.

The strongest manuscript narrative is not that one model is universally best. The benchmark supports the opposite claim: no single model family dominates. Conventional ML, ensembles, TabPFN, MapLight plus GNN transfer, CFA fusion, and Chemprop each win at least one dataset. This is a useful result for an AutoQSAR methods paper because it justifies a flexible model-search workflow rather than a single preferred architecture.

The current run is publication-informative but not yet fully leaderboard-ready. The main gap remains multi-seed TDC evaluation. The current benchmark appears to be a single configured run rather than the TDC-standard 5-seed mean +/- std evaluation.

The per-dataset best-model breakdown strongly supports the "no single model dominates" narrative.

| Winning family | Classification wins | Regression wins | Total wins |
|---|---:|---:|---:|

| TabPFN foundation model | 0 | 5 | 5 |
| CFA combinatorial fusion | 1 | 4 | 5 |
| Graph neural networks, Chemprop | 1 | 0 | 1 |

Interpretation:

- Conventional ML remains highly competitive, especially CatBoost, XGBoost, random forest, AdaBoost, and logistic regression variants.
- Ensembles are especially valuable for classification tasks, winning 11 of 22 classification datasets.
- TabPFN is strongest on smaller regression tasks, including ChemML and FreeSolv-style endpoints.
- MapLight plus GNN transfer is concentrated in regression ADME tasks, especially Polaris and TDC clearance/volume endpoints.
- CFA does not dominate globally, but it produces important wins on specific regression endpoints and one classification task.

## Component Value

The no-rerun ablation analysis decomposes the contribution of major workflow stages across all 44 analyzed datasets.

| Stage added | Datasets improved vs previous stage | Improvement fraction |
|---|---:|---:|
| MapLight classic features | 7 / 44 | 15.9% |

The median marginal improvement is zero for each added stage, which means most datasets do not benefit from every added component. The mean improvements are positive, indicating that the value is concentrated in subsets of datasets. This supports a practical AutoQSAR argument: the workflow is useful because it searches over heterogeneous options and selects what works for a given endpoint.

## Ensemble and CFA Findings

Ensembling is useful but should be presented carefully. It often produces top-3 models, but it does not always beat the best individual base model.

| Ensemble method | Task type | Wins | Top-3 datasets | Beats best base |
|---|---|---:|---:|---:|
| OOF stacking | Classification | 7 / 22 | 12 / 22 | 9 / 22 |
| Inverse-RMSE weighted average | Classification | 4 / 13 | 7 / 13 | 6 / 13 |
| CFA fusion | Classification | 1 / 22 | 6 / 22 | 5 / 22 |
| OOF stacking | Regression | 0 / 22 | 8 / 22 | 2 / 22 |
| Inverse-RMSE weighted average | Regression | 1 / 9 | 7 / 9 | 3 / 9 |
| CFA fusion | Regression | 4 / 22 | 6 / 22 | 4 / 22 |

The most defensible statement is that ensembles are a high-value classification strategy and often remain competitive even when they do not win. CFA is more selective: it is not a universal improvement, but it creates distinct wins that would be missed by conventional ML alone.

## Dataset Difficulty Signals

The dataset difficulty analysis found no single descriptor that strongly explains performance. Dataset size alone has little relationship with estimated leaderboard rank. The strongest observed rank-related signal was selected feature count, with a Spearman correlation of 0.365 against estimated rank across 37 leaderboard-comparable datasets. This may indicate that datasets requiring larger selected feature sets are intrinsically harder or less cleanly represented by the current feature space.

Observed patterns:

- Large classification datasets were mostly won by ensembles.
- Small regression datasets were often won by TabPFN.
- Mid-sized ADME regression datasets were often won by MapLight plus GNN transfer.
- Conventional ML remained competitive across several dataset sizes and both task types.

These trends are useful for a manuscript discussion, but they should be framed as empirical patterns from this benchmark run rather than hard rules.

## Publication-Readiness Caveats

1. The current results appear to be single-run results. TDC leaderboard submission and direct comparison still require 5-seed evaluation with official `admet_group` splits and mean +/- std reporting.
2. Leaderboard comparisons are useful for manuscript positioning, but final claims should verify metric and split compatibility for each benchmark source.
3. Feature-family importance is selector-based. It should be described as selected-feature representation unless model-native importances are added consistently across tree models.
4. Cost tables currently rely mostly on elapsed stage timings from the completed run. The runner changes for inference-only timing and exact AutoQSAR neural parameter counts will make future cost tables stronger.
5. `tdc_herg_central` is incomplete in this run and should be rerun or excluded explicitly.
